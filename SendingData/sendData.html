Basic Overview of Client/Server Process

  • The client, for example a web browser, sends a request to a server. Usually this is a web server like Apache, Nginx, IIS, etc.
  • The server uses the HTTP protocol to process the request.
  • The server then sends a response, also using the HTTP protocol, back to the client.
  • In other words, the HTML form is an easy, user friendly way to use an HTTP request to send the form data to the server.

How to Send Data on the Client Side

  • The <form> element defines how the data will be sent.
  • The attributes used with the <form> element can be configured to request what is to be sent when the submit button is clicked.
    • The action and method attributes are the most important.

The action attribute

  • This attribute defines where the data gets sent.
  • It must be a valid URL to work.
  • If a URL is not present, the data is sent to the URL of the page containing the form.
  • This is an absolute URL where data can be sent: <form action="http://foo.com">
  • This is a relative URL, wherein data is sent to a different URL on the server: <form action="/somewhere_else">

The <method> attribute

  • This defines how data is sent.
  • HTTP protocol provides several ways to make a request.

      These two are the most popular:

    1. GET
    2. POST
  • HTTP protocol: The browser sends a request(to a URL) each time a resource is wanted.
  • This request consists of two parts
    1. header: contains a set of global metadata about the browser's capabilities
    2. body: contains information for the server to process the specific request.

    • The GET method

    • GET is used by the browser to send back a given resource.
    • When a form is sent using the GET method, the data sent to the server is appended to the URL.
