Form Validation

Helpful hints appear to guide users on how to properly format the data they enter. This ensures that when the data is submitted it will work with the application.

Examples of Form Validation:

This field is required(Can't be left blank)

Please enter your phone number using the xxx-xxx-xxxx format.

Please enter a valid email address.(person@peoples.com)

Passwords should be between 9 and 30 characters, contain at least one uppercase letter, at least one symbol, and at least one number.

Different Types of Form Validation

Client-side validation

  • occurs in the browser
  • user friendly, instant resonse
  • Javascript validation is customizable
  • Built-in form validation using HTML5. Has better performance, but is not a customizable as Javascript form validation.

Server-side validation

  • This occurs on the server, after the data has been submitted.
  • Code is used server side to validate the data before it is sent to a database. If it fails, a response is sent back to the client which tells the user which corrections to make.
  • Not as user friendly as client side validation. It requires the entire form be sent to the server, before any errors are submitted to the user.
  • Does provide a more secure way of dealing with data. Many server-side frameworks have built in validation and security features.
  • Developers often use a combination of these two in real world applications.

Built-in-form validation(HTML5)

  • No scripts required!
  • HTML5 form elements can use validation attributes to specify rules for form input.
    • no blank values
    • min/max lengths of data
    • number only
    • pattern matching

    • An element is valid if:

    • The element matches the :valid CSS pseudo-class, which lets you apply a specific style to valid elements.
    • the user sends the data and the browser submits the form, and nothing stops it(e.g. Javascript).

    An element is invalid if:

    • the element matches the :invalid CSS pseudo-class, which lets you apply specific styles to invalid elements.
    • the user sends the data, and the browser bloks the form and shows an error message.
