Input Constraints w/ Pattern(regex)

Validating <input> with HTML5

Ex. Choose your favorite junk food. There are two choices: pretzels or chips. There will be a text <input> with a matching label, and a submit <button>

Validating using the regular expression

  • uses the pattern attribute
  • pattern expects a regex as its value
  • regex is used to match pattern combinations in texxt strings
  • ideal for form validation

    • some regex example patterns:

    • [Ll].*k

      A single char. that is upper or lowercase L, followed by zero or more chars of any type, followed by a single lowercase l

      • [A-Z][A-Za-z' -]+
    • A single uppercase char. followd by one or more chars that are upper or lowercase letters, a dash, an apostraphe, or a space. This could be used to validate the city name of English speaking countries, which start with a capital letter and may contain a dash, space, or apostraphe, but no other special chars.
    [0-9]{3}[ -][0-9]{3}[ -][0-9]{4}
    • A simple match for a US domestic phone number - three digits, followed by a space or dash, followed by three digits, followed by a space or dash, followed by four digits. This can be more complex if you want to add parantheses...
