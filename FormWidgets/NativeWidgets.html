Native Form Widgets

This sectiom will focus on the different types of form widgets. We will be looking at the options for collecting different types of data.

All of the widgets we will be looking at are built into browsers. Keep in mind that these may differe from browser to browser. Custom form widgets can also be built, but that will be covered in a later section.

Form widget common attributes:

  • autofocus

    Default value is FALSE. Puts focus on element when the page loads. Changes when user places focus on a different element.

  • disabled

    Default value is FALSE. Shows that the user can not act with this element.

  • form

    Indicates the form element the widget is associated with. The value of the attribute must be the id attribute of a <form> elment in the same document.

  • name

    The name of the element; this is submitted with the form data.

  • value

    The element's initial value.

Text Input Fields

Text <input> fields are an easy way to let the user enter any type of data. Ceratin text fields can be created for specific data types.

Common Behaviors

  • Can be created as readonly(cannot be modified by user) or disabled(even if data is entered, it is not sent with the rest of the form data)

  • Can have a placeholder, which is text that appears inside the text input area and describes it's purpose.

  • Can be given a certain size(this indicates the size of the box).

    Can be given a maximum length(which is the max number of characters that can be entered in the box.

  • Can be used by spell checkers, if supported by the browser.

Single Line Text Fields

These are created using <input> elements with the type attribute set to text(which is also the default value).

Email address field

Uses the value email for the type attribute.

Email:

When this type is used, the user is required to input a proper email format, or it will issue an error. This is also known as client side verification because it is handled by the browser.

The multiple attribute is used to let the user enter multiple, comma separated, email addresses.

Password field

Uses the value password for the type attribute.

Password

This input will not show what is entered into the field. It instead will show dots or asteriks so that it cannot be read by others.

Security Note:
This will still send the data as plain text, which is a security no-no. Best practice prevention is to host the pages involving forms over a secure htts connection. This will encrypt the data before it is sent.

Search field

The value search is used as the type attribute.

Search:

Search fields are often rendered with rounded corners for styling. They may also be given a button to clear the search field. The search field values can be automatically saved to be auto completed across multiple pages on the same site.

Phone number field

The value tel is used as the type value attribute.

Phone:

Different virtual keypads may be presented on mobile devices. There are no constraints on the value, letters and numbers may be used.

URL fields

The url type is used for the type attribute.

URL:

This adds special validation constraints to the field, with the browser reporting an error if the invalid URLs are entered.

Note:

In most text areas the browser will give a drag handle for resizing on the box. This is found in the lower right hand corner of the box. This can be turned of by setting the text area's resize property to none using CSS.

Other <textarea> attributes

  • cols(default: 20)

    The width of the area, using average charater widths.

  • rows

    The number of visible text lines for the control.

  • wrap(default: soft)

    Indicates how the control wraps text. soft or hard can be used.

    • Notes:

    The <textarea> element is an empty element, which means it can have no child elements.

    The <textarea> element is a regular element and can contain text content children

      Key points for the regular element and text content children:

    1. Instead of defining a value attribute, like with the <input> element, for <textarea> the default text is entered between the start and end taggs.

    2. The <textarea> element only accepts text content. Any HTML content put inside the <textarea> is rendered as if it was plain text.
