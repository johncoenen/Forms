Drop Down Widgets

Lets the user select from a list of options. Benefits UI becuase they do not take up much space.

There are two HTML options for drop downs:

  • select box

  • autocomplete box

Select Box

To create a select box use the <select> element with one or more <option> elements as its children, each with a specific value.

The default value is set by using the selected attribute on the <option> element. This preselects that element when the page loads.

<optgroup>

<optgroup> is used when you want to nest elements to create groups of values

If an <option> element is set with a value attribute, that value will be sent on submission. If value is not used, the <option> element is used as the select box's value. If the label attibute is set on <optgroup>, it may look like a selectable option, but it is not.

Multiple Choice

Add the multiple attribute to the <select> element to enable the user to select several options by using the OS's default keyboard shortcut(Often holding |cmd + ctrl| or holding down |shift| and clicking multiple values will work).

Autocomplete

<datalist>

The <datalist> element can be paired with some child <option> elements to specify the values to display. These provide suggestions for the user.

The data list is then bound to a text field using the input element on the list attribute. When the user begins to enter text in the field, the autocomplete will suggest an option if viable.

EXAMPLE:

<datalist> elements are relatively new to HTML forms, so browser support may vary. A good fallback for this scenario is to wnclose the <option> elements within a <select> element inside the datalist element. Browsers that have support will ignore anything but the <option> elements, and will work. If the browser doesn't support <datalist> it will display the label and the select box.

Checkable Items

The state is changed when clicked on. There are two types: the checkbox and the radio button. Keep in mind that these work a bit differently than other widgets. The big difference is checkable items values are only sent if they are checked. If they are not checked nothing is sent. Not even their name.

For usability purposes enclose each list of related items in a <fieldset> elemen which includes a <legend> that describes the list. Also each <label>/<input> elements should be contatined in its own list item. Values need to be provided for each type of input inside the value attribute. If no value is specified the value on is used.

Check Box

If the checked attribute is included it auto-checks that box on page load.

Radio Button

Multiple radio buttons can be grouped together. If they have the same name attribute, they are considered in the same group of buttons. Only one button per group may be checked in this case. When one is checked all others become unchecked. When the form is sent only the value of the checked button gets sent. If none are checked it and the whole group are considered to be in an unknown state, and no value is sent.

Buttons

There are three types of HTML buttons for forms:

  • Submit

    Sends the form data to the server.

  • Reset

    Resets all form widgets to the default values.

  • Anonymous

    Buttons that have no automatic effect but can be customized using JavaScript code. If you omit the type attribute, this is the default value.

Buttons are created using the <button> element or an <input> element. It's the value of the type attribute that specifies what kind of button is displayed.
